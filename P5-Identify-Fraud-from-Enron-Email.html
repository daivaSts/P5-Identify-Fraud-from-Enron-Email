P5: Identify Fraud from Enron Email¶

1. Introduction & Objective of the Analysis¶

In 2002, Enron, one of the largest companies in the United States, had collapsed into bankruptcy due to widespread corporate fraud. The dataset was originally made public by the Federal Energy Regulatory Commission during its investigation.

The goal was to perform data analysis on Enron dataset to identify Enron employees, who may have committed fraud. I put my machine learning skills to build, tune, evaluate the best performing algorithm, by trying various combinations of supervised classification algorithms and parameters, engineer the features, draw conclusion to make sense of the analysis process and communicatie it to others.

2. Dataset Overview¶

2.1 Exploring the Outliers¶

The dataset contains data of 146 records, with financial and email information organized into 21 features for each. It is not a large dataset to get very clear analysis for a machine learning. The 'poi' feature records whether the person is a person of interest. The dataset contains 18 known 'poi'. To proceed with the analysis, the first feature is separated and put into its own label/target list, and the rest in features list.

  • Scaling features with MinMaxScaler(). The estimator would not be able to learn from new features correctly as expected, since these are orders of magnitude smaller than others, for example total_payments and fraction_to_poi.

  • Using Principal component analysis PCA to reduce the dimensions of a the dataset by projecting it onto a less-dimensional subspace in order to increase the computational efficiency while retaining most of the information. PCA replaces original variables with new variables, called principal components, which are orthogonal, and have variances in decreasing order. The attribute explained_variance_ratio helps to select the optimal number of components. It is a ratio of variance explained by each of the selected components. 3. Algorithm selection¶

Algorithm selection is very important step in the analysis. Since I don't have a lot of data and data are labeled, I should use a supervised learning classifier with high bias. My goal is to create a model that predicts the value of a target variable by learning decision rules inferred from the training data, where each data point is a pair consisting of an input object (typically a vector) and a desired output value.

These are the algorithms I tried to create my model.

  • DecisionTreeClassifier - supervised learning method used for classification and regression. See the metrics Evaluation 1
  • GaussianNB - implements the Gaussian Naive Bayes algorithm for classification. See Evaluation 2
  • AdaBoostClassifier - the principle of AdaBoost is to fit a sequence of weak learners (i.e., models that are only slightly better than random guessing, such as small decision trees) on repeatedly modified versions of the data. AdaBoost can be used both for classification and regression problems. See Evaluation 3
  • RandomForestClassifier - includes two averaging algorithms based on randomized decision trees: the RandomForest algorithm and the Extra-Trees method. See Evaluation 4
  • Support vector machines SVM - a set of supervised learning methods used for classification, regression and outliers detection. See Evaluation 5

These algorithms I picked for the final run, using parameter tuning and validation functions:

  • GaussianNB Classifier ( new features included in the dataset ). See Evaluation 6.
  • DecisionTreeClassifier ( new features included in the dataset ). See Evaluation 7.
  • DecisionTreeClassifier ( new features excluded from the dataset ).See Evaluation 8.

4. Evaluation¶

4.1 Parameter Tuning¶

Parameter tuning is a process of trying and testing various parameters which are passed as arguments to the constructor of the estimator classes.It is important because it helps to find better algorithm. To start using an algorithm, we need beforehand to set parameter values, and later try various combinations of these to see which set of the parameters perform the best when training, in order to improve the fit on the test set.

In my project I was using GridSearchCV in SVC and Decision Tree models to performsearch over supplied parameter values (kernel, C, gamma values), and all the possible combinations of values are evaluated and the best combination is returned.

GridSearchCV - Exhaustive search over specified parameter values for an estimator. The parameters of the estimator are used to apply to fit and predict functions, and are optimized by grid-search over a parameter grid.

To compute the metrics using Support Vector Classification:

  • The parameters passed: parameters = {"kernel":['rbf'], "C":[1, 1000],"gamma": [0.0001, 0.0005, 0.001, 0.005, 0.01, 0.1]}
  • The best SVC parameters: {'kernel': 'rbf', 'C': 1, 'gamma': 0.0001}

To compute the metrics using Decision Tree Classifier:

  • The parameters passed: parameters = {"criterion" : ["gini", "entropy"],"min_samples_split": [2,3] }
  • Decision Tree best parameters : {'min_samples_split': 2, 'criterion': 'entropy'}

4.2 Validation¶

Validation is the process to estimate how well the model has been trained using test data. In my project for my final algorithms, I used these functions:

  • cross_validation.train_test_split() - to split dataset into training and testing subsets where I manually changed the proportion of the dataset to include in the test split. When trying variety of classifiers, best performance was obtained with test_size value 0.35. See Evaluation_metrics of 5 tested classifiers.

  • cross_validation.KFold() - to split and test data into k consecutive folds with various values of the fold parameter. Each fold is used a validation set once while the remaining folds form the training set. For the final 2 algorithms I tried k = (4,5,6) in combination with other parameters. See Final evaluation_metrics for the metrics. GaussianNB Classifier was best with the fold = 4, DT Classifier performed best with fold = 6.

  • SelectKBest() - to select k number of highest scored features. In my project the best performance was achieved when k = 9. See Final evaluation_metrics for the metrics.

  • PCA() - to reduce the feature dimensions using Principal component analysis PCA. It decomposes a multivariate dataset in a set of successive orthogonal components that explain a maximum amount of the variance and bring out strong patterns in a dataset. In PCA I tried values of n_components = (5,7,10), as the maximum number of features to keep. Best performance was achieved with n_components = 8.

  • FeatureUnion() - to concatenate results of estimator from PCA and Univariate SelectKBest selection. It allows to use multiple feature selection processes in parallel.

4.3 Picking Metrics¶

  • The accuracy_score function computes the accuracy, either the fraction or the count of correct predictions from all predictions made.

  • The recall is the ratio tp / (tp + fn) where tp is the number of true positives and fn the number of false negatives. The recall is intuitively the ability of the classifier to find all the positive samples. In this project, recall is the ratio of POI's recognised by the model and all POI's that existed in the dataset. The best value is 1.

  • The precision is the ratio tp / (tp + fp) where tp is the number of true positives and fp the number of false positives. The precision is intuitively the ability of the classifier not to label as positive a sample that is negative. That is, in my project, the precision shows the ratio of recognised POI's and of all the people picked out by a model as POI. The best value is 1.

  • The F1 score can be interpreted as a weighted average of the precision and recall. Best value is 1. The relative contribution of precision and recall to the F1 score are equal. The formula is: F1 = 2 (precision recall) / (precision + recall).
